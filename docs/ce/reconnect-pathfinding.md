# 자동 복구와 패스파인딩

서버 응답 멈춤, 접속 끊김, 로그인 정체 상황에서 복구를 보조하는 선택 기능입니다. Razor 스크립트를 로그인 시 실행으로 설정해두면 서버 점검 끊김을 감지한 후 재접속 시도 -> 서버가 켜지면 재접속 후 자동으로 매크로 복귀 등의 동작도 가능합니다.

## 켜는 위치

`Options → GGO Custom → 서버 프리징 감지`

`0`이면 꺼짐입니다.

## 주요 기능

- 프리징 감지 후 재접속 시도
- 서버 점검/리붓 중 대기 후 재접속
- 로그인 정체 시 재시도
- 반복 실패 시 클라이언트 재시작
- 연결 끊김 팝업에서 멈추지 않고 복구 흐름 진행
- Discord 웹훅 알림

## 패스파인딩 개선

목적지가 막힌 경우에도 갈 수 있는 지점까지 이동합니다. 경로 계산이 오래 걸려 클라이언트가 멈추지 않도록 보호 장치도 들어 있습니다. 기본 CUO 의 개복치 패스파인딩의 성능과 안정성을 매우 개선했습니다.

## 주의 사항

- 자동 복구는 기본 꺼짐입니다.
- 먼저 낮은 위험 환경에서 테스트하세요.
- 웹훅 주소는 개인 채널 정보이므로 공개 제보에 그대로 올리지 마세요.
