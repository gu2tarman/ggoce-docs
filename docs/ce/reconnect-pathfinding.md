# 자동 복구와 패스파인딩

자동 복구 시스템은 장시간 사냥이나 다중 클라이언트 운영 중 서버 응답 멈춤, 접속 끊김, 로그인 정체 상황에서 복구를 보조합니다.

## 원본 CUO와 차이

원본 CUO에서는 접속이 끊기거나 서버 응답이 멈추면 사용자가 직접 재접속해야 합니다. GGO CE는 선택 기능으로 프리징 감지와 자동 재접속 흐름을 제공합니다.

## 켜는 위치

`Options → GGO Custom → 서버 프리징 감지`

값이 `0`이면 꺼짐입니다. 예를 들어 `10`을 입력하면 감지 기준을 두고 자동 복구를 시도합니다.

## 주요 기능

### 프리징 감지

서버 응답이 멈춘 것으로 판단되면 자동으로 재접속을 시도합니다.

### 서버 점검/리붓 대기

서버가 내려간 상황에서는 즉시 실패 처리하지 않고, 서버가 살아날 때까지 기다렸다가 다시 접속합니다.

### 로그인 정체 복구

계정 확인 단계 등에서 로그인이 멈추면 재시도합니다. 반복되면 클라이언트 재시작까지 시도합니다.

### 연결 끊김 팝업 우회

게임 중 연결이 끊겨 팝업에서 멈추는 상황을 줄이고, 즉시 재접속 흐름으로 이어갑니다.

### Discord 웹훅 알림

선택적으로 복구 과정을 Discord 웹훅으로 받을 수 있습니다. 웹훅 주소를 넣은 뒤 테스트 발송으로 확인합니다.

## 패스파인딩 개선

목적지가 막혀 있어도 갈 수 있는 곳까지 이동하도록 개선했습니다. 또한 경로 계산이 과도하게 오래 걸려 클라이언트가 멈추는 상황을 줄이기 위해 보호 장치를 추가했습니다.

## 주의 사항

- 자동 복구는 기본 꺼짐입니다.
- 전투 중 예상치 못한 재접속이 부담되는 캐릭터는 먼저 낮은 위험 환경에서 테스트하세요.
- 웹훅 주소는 개인 채널 정보이므로 공개 제보에 그대로 올리지 마세요.
