# 피드백 보내기

피드백은 종류별로 나누어 보내면 처리 속도가 빨라집니다.

## 버그 제보

버그는 GitHub Issue Form으로 받는 것을 권장합니다.

필수 정보:

- GGO CE 버전
- 런처 버전
- Windows 버전
- 재현 순서
- 기대한 동작
- 실제 동작
- 스크린샷 또는 로그

## 기능 제안

기능 제안은 바로 개발 이슈로 만들기보다, 먼저 사용 상황과 기대 효과를 설명해 주세요.

좋은 제안 예시:

- 어떤 상황에서 불편한지
- 지금은 어떻게 우회하고 있는지
- 기능이 생기면 어떤 흐름이 줄어드는지

## 질문

단순 사용 질문은 오픈카톡, 디스코드, GitHub Discussions 같은 대화형 채널이 좋습니다. 반복되는 질문은 FAQ로 옮깁니다.
